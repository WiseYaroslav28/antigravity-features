# AntiGravity Features Service

Этот проект представляет собой **изолированный MCP-сервер**, разработанный для расширения возможностей, автоматизации и локализации интерфейса приложения AntiGravity v2.1.4 (и выше).

## Основной функционал

### 1. Локализация интерфейса (русский язык)
* В приложении реализован механизм **Hard Patching** системного скрипта `preload.js` внутри `app.asar`.
* Перевод осуществляется по событиям отрисовки DOM с помощью `MutationObserver`, который на лету заменяет текстовые узлы (`characterData` и `childList`).
* Решение работает мгновенно перед отрисовкой страницы, предотвращая мигание оригинального текста (FOUC).
* Имена файлов, пути и названия артефактов автоматически защищены от перевода благодаря функции `isFileName`.

### 2. Система автосохранения и восстановления настроек (Backup & Restore)
Сервер автоматически решает проблему сброса настроек и размеров окон при сбоях или переустановке AntiGravity:
* **Window State:** Регулярно бэкапит файл `window-state.json` (координаты, размеры окон, статус развертывания).
* **App Storage:** Создает резервные копии `app_storage.json` (настройки фонового запуска, предотвращения спящего режима и т.д.).
* **Global Permissions:** Бэкапит `config.json` (разрешенные пути, правила автовыполнения терминала и настройки безопасности агента).
* **Project Permissions:** Бэкапит настройки разрешений для каждого отдельного проекта в папке `projects/`.

**Как это работает:**
* Каждые 30 секунд сервер сканирует файлы в `AppData\Roaming\Antigravity\` и `.gemini\config\`. Если они содержат валидные JSON-данные и изменились, они копируются в локальную папку `backups/` внутри проекта.
* При старте AntiGravity сервер проверяет целостность этих файлов. Если файлы настроек сбросились, повредились или пропали, они **мгновенно восстанавливаются из последнего бэкапа** еще до того, как Electron прочитает их с диска.

## Структура проекта

* **`index.js`** — Фоновый сервис (NodeJS), осуществляющий мониторинг окон, инжекцию перевода и периодическое резервное копирование файлов настроек.
* **`localization_injected.js`** — Скрипт локализации для Chromium (MutationObserver, словари, регулярные выражения).
* **`package.json`** — Конфигурация NodeJS проекта и список зависимостей.
* **`backups/`** — Папка автоматических бэкапов конфигурационных файлов и размеров окон.

## Как обновлять переводы и настройки?

### Редактирование словаря перевода:
1. Откройте `localization_injected.js` в папке проекта.
2. Измените или добавьте записи в `translationDictionary` (точные совпадения) или регулярные выражения в `regexTranslations`.
3. Запустите скрипт синхронизации: `node update_preload_dicts.js` (он обновит словари внутри распакованного `app_extracted_v2.1.4`).
4. Пересоберите ASAR-архив: `npx asar pack app_extracted_v2.1.4 app.asar.original`.
5. Запустите скрипт `restart_app.cmd` через `wmic process call create "cmd.exe /c restart_app.cmd"` (чтобы избежать убийства процесса), который заменит оригинальный `app.asar` и перезапустит IDE.

Для подробной технической информации и истории принятых архитектурных решений (ADR) см. `docs/project_memory.md`.

## Архитектурный паттерн: Бесшовное автообновление MCP-серверов в Windows без аварийного завершения

На операционной системе Windows файлы запущенных NodeJS-процессов (`index.js`, модули и логи) блокируются системой для записи (`EBUSY`). Чтобы обновить файлы MCP-сервера в рантайме, не вызывая при этом ошибку аварийного завершения процесса (`Error: exit status 1` / красную точку в настройках IDE), необходимо применять **паттерн мягкого отключения**:

### Шаги бесшовного обновления:
1. **Регистрация обработчиков завершения в MCP-сервере:**
   В основном коде MCP-сервера (например, `index.js`) необходимо слушать события `stdin` и завершать процесс с кодом `0` (штатно):
   ```javascript
   process.stdin.on('close', () => process.exit(0));
   process.stdin.on('end', () => process.exit(0));
   ```
2. **Мягкое отключение через конфигурацию в скрипте обновления (update-runner):**
   * **Шаг А:** Скрипт обновления считывает глобальный конфигурационный файл `mcp_config.json`, временно удаляет обновляемый сервер из списка `mcpServers` и сохраняет файл.
   * **Шаг Б:** IDE мгновенно обнаруживает изменение конфигурации и мягко закрывает поток `stdin` для удаленного сервера. Благодаря обработчикам `close` и `end` процесс сервера завершается штатно с кодом `0`.
   * **Шаг В:** Скрипт обновления ожидает 3 секунды (`setTimeout`), пока файлы разблокируются.
   * **Шаг Г:** Выполняется безопасная замена файлов в рантайм-директории.
   * **Шаг Д:** Скрипт обновления восстанавливает конфигурацию сервера в `mcp_config.json` с новым параметром среды (например, `"env": { "RELOAD_TRIGGER": "timestamp" }`). IDE считывает изменения и запускает чистый процесс с новым кодом без ошибок в интерфейсе.

## Рекомендация по разработке

Рекомендуется открыть эту папку (`C:\Antigravity projects\antigravity-features`) как **активную рабочую область (Active Workspace)** в настройках AntiGravity, чтобы вести дальнейшую разработку новых надстроек, управлять бэкапами и отслеживать историю изменений.
