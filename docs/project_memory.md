# Глобальная память проекта: Antigravity Features

## 1. Паспорт проекта
* **Суть проекта:** Набор модификаций и улучшений (фичей) для IDE Antigravity v2.1.4+. Основной упор сделан на глубокую русификацию интерфейса и создание системы бэкапа настроек.
* **Технологический стек:** Node.js, Electron (ASAR), MutationObserver, PowerShell/cmd.
* **Команды запуска:**
  * Извлечение исходников: `npx asar extract app.asar app_extracted_v2.1.4`
  * Обновление словарей: `node update_preload_dicts.js`
  * Сборка ASAR: `npx asar pack app_extracted_v2.1.4 app.asar.original`
  * Перезапуск с заменой: `wmic process call create "cmd.exe /c restart_app.cmd"`

## 2. Архитектура системы
Изначально предполагалось использовать подход с инжекцией скриптов через отладочный порт (CDP - Chrome DevTools Protocol) с помощью отдельного MCP-сервера. Однако этот подход оказался нестабильным и вызывал задержки в переводе (FOUC).

Текущая архитектура основана на **Hard Patching** внутреннего файла `preload.js` в `app.asar`.
* `localization_injected.js` — Исходный файл, содержащий `translationDictionary` (словарь точных совпадений) и `regexTranslations` (правила на основе регулярных выражений).
* `update_preload_dicts.js` — Node.js скрипт, который парсит `localization_injected.js` и заменяет соответствующие блоки внутри `app_extracted_v2.1.4/dist/preload.js`.
* `preload.js` — Системный скрипт Electron, который выполняется в контексте каждого окна перед его загрузкой. В него встроен `MutationObserver`, который на лету подменяет текстовые узлы (`nodeType === Node.TEXT_NODE`).

## 3. Журнал принятых и отвергнутых решений (ADR / Decision Log)

### [ADR-0001] Выбор механизма инжекции перевода
* **Контекст:** Изначально интерфейс Antigravity (React/Vue) загружается на английском языке. Нам требовалось перехватить рендеринг и заменить текст на русский.
* **Принятое решение:** Использование `MutationObserver` для перехвата `characterData` и `childList` изменений текстовых узлов.
* **Отвергнутые альтернативы:**
  * *Поиск и замена в React Bundle (`main.js`):* Отвергнуто, так как бандл минифицирован, подвержен изменениям при малейшем апдейте IDE и сложен в реверс-инжиниринге.
  * *Переопределение `document.body.innerHTML`:* Отвергнуто, так как это полностью ломает привязки обработчиков событий (Event Listeners), устанавливаемые фреймворками.
* **Последствия:** Приходится поддерживать большой словарь точных строк, а динамические строки (например, с цифрами "23 active conversations") обрабатывать через регулярные выражения. Так как элементы DOM могут содержать вложенные теги (`<span>23</span> <span>conversations</span>`), регулярные выражения иногда приходится разбивать на части или адаптировать под структуру DOM-дерева.

### [ADR-0002] Способ доставки кода (CDP vs ASAR Patching)
* **Контекст:** Для запуска `MutationObserver` нужно было доставить скрипт в каждое окно IDE.
* **Принятое решение:** Распаковка `app.asar`, модификация системного `preload.js` и обратная запаковка (`asar pack`).
* **Отвергнутые альтернативы:** *Инжекция через Chrome DevTools Protocol (CDP).* Отвергнуто из-за задержек (текст сначала появлялся на английском, затем моргал и становился русским), а также проблем с доступом к внутренним iframe'ам и всплывающим окнам.
* **Последствия:** Обновление словаря требует пересборки `app.asar` и перезапуска всего приложения. Для автоматизации мы создали скрипт синхронизации словарей `update_preload_dicts.js`.

### [ADR-0003] Перезапуск приложения и обход ограничений (Process Tree Escaping)
* **Контекст:** Для применения нового `app.asar` необходимо перезапустить Antigravity. IDE блокирует файл во время работы.
* **Принятое решение:** Создан `restart_app.cmd`, который убивает процессы IDE, заменяет файл и запускает IDE заново. Запуск самого скрипта производится через `wmic process call create "cmd.exe /c restart_app.cmd"`.
* **Отвергнутые альтернативы:** Запуск скрипта напрямую через терминал или `child_process`. Отвергнуто, так как ИИ-агент (и терминал) является дочерним процессом IDE. При закрытии IDE агент умирал до того, как успевал скопировать файл, из-за чего скрипт обрывался. `wmic` позволяет создать полностью независимый процесс.

### [ADR-0004] Защита от запуска в интернете (`isLocal`)
* **Контекст:** В IDE могут открываться внешние веб-сайты. Наш переводчик не должен ломать их работу.
* **Принятое решение:** В `preload.js` добавлена проверка протоколов. Скрипт инициализируется только если протокол совпадает с `vscode-file:`, `vscode-webview:`, `antigravity-ide:` или если хост равен `127.0.0.1`/`localhost`.
* **Отвергнутые альтернативы:** *Простая проверка `!window.location.protocol.startsWith('http')`.* Отвергнуто, так как некоторые системные панели в IDE могут загружаться по `http://localhost`.
